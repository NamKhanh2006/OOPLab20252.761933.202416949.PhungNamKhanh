Question:
If you create a constructor method to build a DVD by title then create a constructor method to build a DVD by category. Does JAVA allow you to do this? 
Answer:
No, Java will not allow you to do that. A constructor signature, which helps Java tell two constructors apart, is defined by the number, order, and type of its parameters.
If you write:
Constructor 1: public DigitalVideoDisc(String title)

Constructor 2: public DigitalVideoDisc(String category)
From Java's perspective, both of these are exactly the same, because both have one single String argument. If you tried to compile this, the Java compiler 
would give you a "Duplicate method" error because it wouldn't know which one to call when you write new DigitalVideoDisc("Horror"), for example.
